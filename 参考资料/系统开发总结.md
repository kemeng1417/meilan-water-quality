# 美兰机场供水站水质管理系统 — 开发总结

## 项目背景

海口美兰机场供水站原使用 Word 文档手工填报每日水质检测数据，存在以下痛点：

- 数据分散在多个 Word 文件中，无法统一检索和历史追溯
- 超标判断依赖人工对照标准，容易遗漏
- 30+ 个直饮水点位手工填报工作量大、易出错
- 报告格式不统一，审核流程不清晰
- 无法生成趋势分析图表

本系统在完全本地化、离线运行的前提下，实现了水质检测数据填报、自动合规判定、报告生成、趋势分析和异常管理的全流程数字化。

## 技术架构

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 后端框架 | Python 3.13 + FastAPI | 高性能异步框架，适合数据处理和报表生成 |
| 数据库 | SQLite (WAL 模式) | 零配置、免安装、单文件存储 |
| ORM | SQLAlchemy 2.0 | 模型驱动，迁移方便 |
| 前端框架 | React 18 + TypeScript | 组件化开发，类型安全 |
| UI 组件库 | Ant Design 5.x | 中文支持优秀，Table/Form 组件成熟 |
| 图表库 | Apache ECharts | 国内主流，交互丰富 |
| 报告导出 | python-docx + openpyxl | Word/Excel 生成 |
| PDF 导出 | Word COM 自动化 | Windows 原生支持，中文渲染完美 |
| AI 识别 | Qwen-VL-OCR (DashScope) | 拍照自动识别手填检测数据 |
| 打包分发 | PyInstaller + VBS | 单文件 EXE，双击即用 |

## 核心功能模块

### 1. 数据填报（核心页面）
- 矩阵/逐点双模式填报，支持 Tab 跳格、Enter 换行
- 实时合规判定：录入时即时对比 GB 5749-2022 / CJ 94-2005 标准限值
- 超标值红色标记 + 自动结论生成
- IME 输入法保护（防止中文拼音中间态误触发）
- Undo/Redo 支持（Ctrl+Z / Ctrl+Y，最多 50 步）
- 一键填充、批量粘贴、列头快速填充
- 整行删除/添加采样点
- 联合报告（出厂水+末梢水）双限值展示

### 2. 首页看板
- 4 统计卡片（今日/本月/待审核/超标数），点击跳转对应记录列表
- 近 7 天合格率趋势图（ECharts 折线图，自适应 Y 轴）
- 今日各水样类型完成进度
- 本月超标指标 TOP5 排行

### 3. 检测记录管理
- 多条件筛选（水样类型/状态/日期/关键字/超标）
- 状态统计栏（草稿/已提交/已审核/已打回）
- 快捷审核（输入姓名即可审核通过/打回）
- 批量操作（删除/导出）
- 行展开预览检测明细

### 4. 趋势分析
- 双 Y 轴设计（数值型指标 + 百分比/计数型指标分离）
- dataZoom 区域缩放，异常标记点
- 统计卡片（均值、最大值、最小值、超标次数等）
- 数据明细表格联动
- 折线图/柱状图切换

### 5. 采样点管理
- 搜索、排序、区域彩色标签
- 使用统计（检测次数、最近检测日期）
- 批量操作（启停、更改区域）
- 区域/楼层自动补全

### 6. 报告导出
- **Word**: A4 横向，标准限值参考行，照片内嵌，宋体小四
- **Excel**: 冻结表头，绿/红单元格标记合格/超标，每行结论列
- **HTML**: 自包含 base64 图片，响应式打印样式，一键打印 PDF
- **PDF**: Word COM 自动化转换，失败自动降级为 HTML
- 照片随报告导出
- 结论支持手动编辑，编辑后内容同步导出

### 7. 现场照片
- 所有记录均支持拍照上传
- 按采样点分组展示
- 缩略图直接显示，点击全屏大图预览
- 支持批量上传和删除

### 8. 异常告警
- 超标自动创建告警记录
- 告警列表、处理状态跟踪
- 整改措施录入

### 9. 用户与权限
- 2 种角色：化验员（填报+审核）、主管（全部权限+人员管理）
- 互审机制：检测人之间互相审核
- 支持编辑个人姓名、修改密码

### 10. AI 拍照识别（预留）
- 拍照 → Qwen-VL-OCR 提取文字 → 文本模型解析结构化 JSON
- 审核弹窗支持逐个单元格修正
- 一键填充到填报表格
- 指标名模糊匹配（pH↔pH值、COD↔COD(耗氧量)）
- 识别速度约 3-5 秒

## 关键设计亮点

### 1. 合规判定引擎
三种判定模式并行：
- **定性匹配**: 如"总大肠菌群不应检出" → 匹配 "未检出"=合格、"检出"=超标
- **数值比较**: 对比 min/max 限值，自动判断
- **特殊处理**: "合格"标记即视为达标

### 2. 联合报告双限值
出厂水+末梢水联合报告（water_type_id=4）自动加载两份标准限值，限值不同时显示 `出厂:≥0.3 末梢:≥0.05`。

### 3. 自动结论生成
保存明细时自动生成结论：
- 异常项数量、涉及点位、超标指标 → 完整描述
- 合格则生成 "本次检测项目全部合格，符合XX标准" 
- 用户可手动编辑，清空后重新保存会重新生成

### 4. 报告编号规则
`{前缀}-{YYYYMMDD}-{序号}` — CCS(出厂水)/MSS(末梢水)/ZYS(直饮水)/LH(联合)，同日同类型自动递增。

### 5. 离散值填报处理
出厂水每日只测 2-3 个检测点位，通过采样点勾选机制支持离散填报，避免创建大量空白行。

### 6. 离线桌面化部署
- PyInstaller 打包为单文件 EXE
- VBS 脚本隐藏终端窗口
- 桌面快捷方式，双击启动
- 浏览器访问 localhost:8000

## 数据模型

```
水样类型(WaterType) ──┐
                       ├── 检测记录(TestRecord) ── 检测明细(TestDetail) ── 检测指标(Indicator)
采样点位(SamplePoint) ─┘       │
                       ├── 超标记录(AlertRecord)
                       ├── 照片(Photo)
用户(User) ────────────┘
```

### 标准限值覆盖

| 水样类型 | 执行标准 | 指标数 |
|---------|---------|--------|
| 出厂水 | GB 5749-2022 | 9 |
| 末梢水 | GB 5749-2022 | 9 |
| 直饮水 | CJ 94-2005 | 9 |
| 联合报告 | 双标准 | 9 |

### 9 项日检指标
肉眼可见物、浑浊度、色度、pH、COD(耗氧量)、菌落总数、总大肠菌群、游离余氯、臭和味

## 状态流转

```
draft (草稿) → submitted (已提交) → reviewed (已审核)
                    ↓
              rejected (已打回) → draft (重新编辑)
```
